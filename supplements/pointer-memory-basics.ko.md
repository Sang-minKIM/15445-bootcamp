# 포인터, 주소, 메모리 그림 기초

이 문서는 `src/pointer_basics.cpp`, `src/references.cpp`, `src/unique_ptr.cpp` 같은 파일을 읽기 전에
알아두면 좋은 **포인터(pointer)**, **주소(address)**, **객체 수명(object lifetime)** 기초를 설명합니다.

---

## 1. `p`는 pointer의 약자야?

보통은 그렇습니다.

```cpp
int x = 10;
int *p = &x;
```

여기서 `p`는 "pointer"를 줄여 쓴 변수명처럼 자주 쓰입니다.
꼭 `p`를 써야 하는 건 아니고, 아래처럼 더 구체적으로 이름 붙여도 됩니다.

```cpp
int *ptr = &x;
int *x_ptr = &x;
int *score_ptr = &score;
```

---

## 2. 가장 중요한 한 줄

```cpp
int x = 10;
int *p = &x;
```

이 코드는 다음 뜻입니다.

- `x`는 정수 값 `10`을 저장하는 변수
- `&x`는 `x`가 저장된 메모리 주소
- `p`는 그 주소를 저장하는 포인터 변수

즉:

- `p`는 `x`를 저장하지 않고
- **`x`의 주소를 저장합니다**

---

## 3. 표로 정리

```cpp
int x = 10;
int *p = &x;
```

| 표현 | 의미 | 타입 | 예시 값 |
|---|---|---|---|
| `x` | 실제 값 | `int` | `10` |
| `&x` | x의 주소 | `int *` | `0x1000` |
| `p` | x의 주소를 저장한 포인터 변수 | `int *` | `0x1000` |
| `*p` | p가 가리키는 곳의 실제 값 | `int` | `10` |
| `&p` | 포인터 변수 p 자신의 주소 | `int **` | `0x2000` |

중요한 관계:

```cpp
p == &x     // true
*p == x     // true
```

---

## 4. 메모리 그림으로 보기

다음 코드가 있다고 합시다.

```cpp
int x = 10;
int *p = &x;
```

메모리를 아주 단순하게 그리면 이렇게 볼 수 있습니다.

```text
주소        내용
0x1000      10        <- x
0x2000      0x1000    <- p
```

즉:

- `x`는 주소 `0x1000`에 있고, 그 안의 값은 `10`
- `p`는 주소 `0x2000`에 있고, 그 안의 값은 `0x1000`

여기서 `p` 안에 들어 있는 `0x1000`은
바로 **x가 있는 위치**입니다.

그림으로 한 번 더 보면:

```text
p
|
| 저장된 값: 0x1000
v
x
|
| 저장된 값: 10
v
10
```

즉, `p`는 `x`를 "직접" 저장하는 게 아니라,
**x가 어디 있는지 가리킵니다.**

---

## 5. `*p`는 왜 10이야?

`p` 안에는 `x`의 주소가 들어 있습니다.

```cpp
std::cout << p << std::endl;   // 예: 0x1000
```

그리고 `*p`는
"p가 저장한 주소로 가서, 거기 있는 실제 값을 읽어라"는 뜻입니다.

```cpp
std::cout << *p << std::endl;  // 10
```

즉:

```text
p     = 주소
*p    = 그 주소에 있는 실제 값
```

---

## 6. `&`와 `*`를 어떻게 읽으면 돼?

### `&`
주소를 얻는 연산자

```cpp
&x
```

= "`x`의 주소"

### `*`
포인터가 가리키는 실제 값에 접근하는 연산자

```cpp
*p
```

= "`p`가 저장한 주소로 가서 그 값을 보기"

---

## 7. `&x`, `p`, `*p`, `&p`를 같이 보기

```cpp
int x = 10;
int *p = &x;
```

### 값/주소 관계

```text
x   = 10
&x  = 0x1000
p   = 0x1000
*p  = 10
&p  = 0x2000
```

### 메모리 그림

```text
0x2000: 0x1000   <- p
0x1000: 10       <- x
```

그래서:

- `p == &x`
- `*p == x`

입니다.

---

## 8. `&p`는 왜 따로 있어?

`p`도 변수이기 때문입니다.

```cpp
int x = 10;
int *p = &x;
```

이때:

- `x`는 `int` 변수
- `p`는 `int *` 변수

변수라면 자기 자신도 메모리 어딘가에 저장됩니다.
그래서 `p`의 주소도 존재합니다.

```cpp
&p
```

는 "포인터 변수 `p`가 저장된 위치"입니다.

그래서 타입도:

```cpp
p   : int *
&p  : int **
```

가 됩니다.

---

## 9. `*p = 20`을 하면 무슨 일이 생겨?

```cpp
int x = 10;
int *p = &x;
*p = 20;
```

메모리 그림:

```text
수정 전
0x1000: 10       <- x
0x2000: 0x1000   <- p

수정 후
0x1000: 20       <- x
0x2000: 0x1000   <- p
```

즉 `p`가 바뀐 게 아니라,
**p가 가리키는 곳의 값이 바뀐 것**입니다.

그래서:

```cpp
std::cout << x << std::endl;  // 20
```

가 됩니다.

---

## 10. 객체 수명(object lifetime)은 뭐야?

객체 수명은
**그 객체가 언제 생성되고, 언제 사라지는지**
를 뜻합니다.

예:

```cpp
int main() {
  int x = 10;
}
```

`x`는 `main` 안에서 생성되고,
`main`이 끝나면 사라집니다.

이걸 그림으로 보면:

```text
main 시작
  x 생성
main 종료
  x 소멸
```

---

## 11. lifetime이 왜 중요해?

포인터는 주소만 저장합니다.
그런데 그 주소가 가리키는 객체가 이미 사라졌다면 문제가 됩니다.

```cpp
int *bad_pointer() {
  int x = 10;
  return &x;   // 위험
}
```

이 함수 안에서:

- `x`는 함수 안에서만 존재
- 함수가 끝나면 `x`는 사라짐
- 그런데 `&x`를 반환함

즉 반환된 포인터는
**이미 사라진 객체의 주소**를 들고 있게 됩니다.

그림으로 보면:

```text
bad_pointer 함수 내부
0x1000: 10   <- x

함수 종료 후
x는 사라짐

반환된 포인터는 여전히 0x1000을 들고 있음
하지만 그 위치의 값은 더 이상 안전하게 사용할 수 없음
```

이런 포인터를 보통 **dangling pointer**라고 부릅니다.

## 12. 왜 뒤 파일들이 이걸 안다고 가정해?

### `references.cpp`
- 참조는 "같은 객체를 다른 이름으로 다루는" 개념이라
- 메모리에서 같은 대상을 가리킨다는 감각이 필요합니다.

### `iterator.cpp`
- 반복자는 내부적으로 "어느 원소를 가리키는지" 감각이 중요합니다.

### `unique_ptr.cpp`, `shared_ptr.cpp`
- "누가 이 객체를 소유하나?"
- "언제 파괴되나?"
- 가 핵심이라 lifetime 이해가 꼭 필요합니다.

### `move_semantics.cpp`
- 자원과 소유권이 다른 객체로 넘어가는 개념이라
- 값, 주소, 객체 수명 감각이 필요합니다.

---

## 13. 초반에는 이렇게 이해해도 충분함

처음에는 아래 정도만 확실히 잡으면 됩니다.

1. 변수는 메모리 어딘가에 저장된다.
2. 주소는 그 위치다.
3. 포인터는 주소를 저장한다.
4. `*포인터`는 그 주소에 있는 실제 값이다.
5. 객체가 사라진 뒤 그 주소를 계속 쓰면 안 된다.

---

## 14. 가장 최소한의 예제

```cpp
#include <iostream>

int main() {
  int x = 10;
  int *p = &x;

  std::cout << "x: " << x << std::endl;
  std::cout << "&x: " << &x << std::endl;
  std::cout << "p: " << p << std::endl;
  std::cout << "*p: " << *p << std::endl;

  *p = 25;
  std::cout << "x after *p = 25: " << x << std::endl;
}
```

이 코드에서 핵심은:

- `p`와 `&x`는 같은 주소값
- `*p`와 `x`는 같은 실제 값

이라는 점입니다.

---

## 15. 한 줄 요약

```cpp
int x = 10;
int *p = &x;
```

이 코드는

> "`p`는 `x`의 주소를 저장하는 포인터이고, `*p`를 통해 `x`에 접근할 수 있다"

는 뜻입니다.

그리고 여기에 lifetime까지 더하면:

> "포인터는 주소를 저장할 뿐이므로, 그 주소의 객체가 아직 살아 있는지도 항상 중요하다"

가 됩니다.
